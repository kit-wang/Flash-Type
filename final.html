Project Overview/Description

Hello! This is the main landing page for Flash Type. Our team, Kit Kat Juice, aims to create a Python program that functions as a typing speed test (with a GUI), which will test how fast the user can type a random paragraph (created through a program like generate_text and displayed one line at a time) based on characters/words per minute. At the start of each “round” of the typing test will have a mini-section that allows the user to choose the next text they will be tested on. At the end of the round, should the user choose to end it, error frequency analysis and the typing speed computations will be displayed (this will be made possible with the `time` module). The random sentences generated for the user to test themselves on are taken from various text files from Project Gutenberg.

Instructions

A typing test measures the test-taker's typing speed and accuracy. Typing tests usually give one or more paragraphs for the typer to copy within a certain time frame. Based on how fast the tester is able to copy that passage, and the amount of errors they make while typing it, is summed up as results at the end of the typing test. Flash Type is a specific typing test that will utilize data analysis and imported modules using Python. To play, the program's first line will have the words "Type the following line: " The words after that are the sentence(s) that the tester will need to type out. The sentence afterwards will be "press ENTER when you are ready to test :)", prompting the user to press the enter key to begin the countdown. Once you finish typing the sentence, click enter to show you are finished. Afterwards, the next line will show how many errors you have, and the line after that will show how many seconds it took to type the sentence. This process will continue multiple times to test various sentences. To sum up the directions:

  1. Look at the sentence next to "Type the following line: "
  2. Press enter to start typing.
  3. Type the sentence above to the best of your ability. Press enter again when done.
  4. The program will show you the number of errors and time taken. The next sentence to practice typing will also be provided.
  5. Repeat steps until the typing test is over.

How it Works

Flash Type will take a randomly generated sentence from one of the chosen texts (see the Files section for more info on the texts used). The user will retype that sentence. The program will assess how correct the user's sentence was (such as any errors they made when compared to the original sentence). The Time module and program calculations will be based on one line at a time, and this data is then scaled to record wpm (words per minute).

Resources

Flash Type uses two Python modules: Pygame and time.

  • Pygame: The Pygame library was implemented so that there is a user-friendly interface for the program. Since Pygame is generally used for developing games in Python, implementing Pygame into Flash Type would help make the typing test more user-friendly. To see the official Pygame documentation repository, click here. Also, this website has a general overview of Pygame.
  • time: The time module will help with determining calculations, such as typing speed, after each round of typing ends. This allows the results at the end of the typing test to be more accurate, such as assessing how much time elapsed throughout the typing rounds. This website gives an overview of the time module.
  • Statistics: This library was imported to find the mode of the erorr list and to find the most frequent error. Information about the statistics can be found here.

Bugs/Errors/Things to be Aware of

  • Since the sentences are randomly generated, this means that the length of the sentences for each round can vary. While this is to create for a more varied test, this also means some sentences may be of shorter or longer length than others.
  • The timer allows the user to go a few seconds over the time limit (this can lead to slightly imprecise data, but is compensated).
  • For every round, the user currently has to click on the text box before typing the line in order for the calculations to be accurate. Otherwise, the final WPM outcome may show as 0.
